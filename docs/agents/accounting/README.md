# Робоча папка бухгалтера Calendary

## Про роль

Бухгалтер відповідає за фінансовий облік, звітність, податки та контроль за грошовими потоками в системі Calendary.

---

## Основні обов'язки

### 1. Фінансовий облік
- Облік всіх транзакцій (продажі, витрати)
- Контроль платежів через Monobank
- Звірка банківських виписок
- Управління дебіторською заборгованістю

### 2. Податкова звітність
- ЄСВ (Єдиний соціальний внесок)
- ПДВ (Податок на додану вартість)
- Податок на прибуток
- Звіт про фінансові результати
- Єдиний податок (якщо ФОП на спрощеній системі)

### 3. Управління витратами
- Облік витрат на друк (договори з друкарнями)
- Вартість доставки (Нова Пошта)
- Маркетингові витрати (реклама)
- Операційні витрати (хостинг, софт, зарплати)

### 4. Аналітика та звітність
- Прибутковість по продуктах
- Маржинальність замовлень
- ROI маркетингових кампаній
- Cash flow прогнозування
- Фінансові KPI

## Структура доходів та витрат

### Доходи

#### B2C (Індивідуальні замовлення)
```
Приклад:
Календар настінний A3: 350 грн
- Собівартість друку: 80 грн
- Доставка НП (оплата клієнтом): 0 грн
- Маржа: 270 грн (77%)
```

#### B2B (Корпоративні замовлення)
```
Приклад:
100 календарів настільних:
Ціна за одиницю: 200 грн
Загальна сума: 20,000 грн
- Собівартість друку: 50 грн * 100 = 5,000 грн
- Доставка (одна адреса): 150 грн
- Знижка 15%: -3,000 грн
- Маржа: 11,850 грн (59%)
```

#### Цифрові продукти (PDF)
```
Приклад:
Календар PDF для самостійного друку: 150 грн
- Собівартість: ~0 грн (тільки генерація)
- Маржа: 150 грн (100%)
```

### Витрати

#### Прямі витрати (Cost of Goods Sold)
- **Друк** - оплата друкарням
- **Матеріали** - папір, біндінг
- **Доставка** - послуги Нової Пошти (якщо не оплачує клієнт)

#### Операційні витрати (OPEX)
- **Маркетинг** - реклама (Facebook, Google, Instagram)
- **Хостинг** - серверні витрати
- **Софт** - ліцензії, підписки
- **Зарплати** - команда
- **Комісії** - Monobank (1-2% від транзакції)

#### Адміністративні витрати
- Бухгалтерські послуги
- Юридичні послуги
- Банківське обслуговування

## Податкова система

### Варіант 1: ФОП на спрощеній системі (3 група)
**Податки:**
- Єдиний податок: 5% від доходу (або 3% з ПДВ)
- ЄСВ: 22% від мінімальної зарплати

**Обмеження:**
- Річний дохід до 7 млн грн
- Не більше 10 найманих працівників

**Переваги:**
- Проста звітність
- Низькі податки
- Легко почати

### Варіант 2: ТОВ на загальній системі
**Податки:**
- Податок на прибуток: 18%
- ПДВ: 20% (з можливістю відшкодування)
- ЄСВ: 22% від зарплати

**Переваги:**
- Без обмежень по доходу
- ПДВ відшкодування
- Краща репутація для B2B

## Інтеграція з системою Calendary

### API endpoints для бухгалтерії

```typescript
// Отримання фінансових звітів
GET /api/accounting/reports/p-and-l?from=2025-01-01&to=2025-01-31

// Експорт транзакцій
GET /api/accounting/transactions/export?format=csv

// Звіт по витратах
GET /api/accounting/expenses?category=marketing

// Звірка з Monobank
GET /api/accounting/reconciliation/monobank
```

### Автоматизація

**Що можна автоматизувати:**
1. Імпорт банківських виписок з Monobank
2. Розпізнавання платежів (matching з замовленнями)
3. Формування рахунків для корпоративних клієнтів
4. Експорт даних в 1C/BAS
5. Щоденні/тижневі email звіти

**Приклад автоматичної звірки:**
```typescript
// Щоденна задача (cron job)
async function reconcilePayments() {
  // 1. Отримати транзакції з Monobank за вчора
  const bankTransactions = await monobank.getTransactions(yesterday);

  // 2. Звірити з замовленнями в системі
  for (const transaction of bankTransactions) {
    const order = await findOrderByAmount(transaction.amount);

    if (order) {
      // Оновити статус оплати
      await updateOrderPaymentStatus(order.id, 'paid');

      // Записати в облік
      await recordTransaction({
        type: 'revenue',
        amount: transaction.amount,
        orderId: order.id,
        date: transaction.date
      });
    } else {
      // Незвірена транзакція - відправити alert
      await notifyAccountant({
        type: 'unmatched_transaction',
        transaction
      });
    }
  }
}
```

## Фінансові KPI

### Revenue Metrics
- **MRR** (Monthly Recurring Revenue) - місячний дохід
- **YoY Growth** - зростання рік до року
- **Revenue per Channel** - дохід по каналах

### Profitability Metrics
- **Gross Margin** - валова маржа (мінімум 60%)
- **Net Margin** - чиста маржа (мінімум 15%)
- **EBITDA** - прибуток до вирахування відсотків, податків та амортизації

### Cost Metrics
- **CAC** (Customer Acquisition Cost) - вартість залучення клієнта
- **COGS** - собівартість продажу
- **Marketing ROI** - ROI маркетингу

### Cash Metrics
- **Burn Rate** - швидкість витрат (для стартапу)
- **Runway** - скільки місяців компанія може працювати на поточних запасах
- **Days Sales Outstanding (DSO)** - середній час оплати

## Робота з друкарнями

### Договори
- Укласти договори з 2-3 друкарнями (резервні варіанти)
- Узгодити ціни (залежно від тиражу)
- Умови оплати (передоплата/постоплата)
- Терміни виготовлення

### Приклад ціноутворення друкарні
```
Календар A3 настінний:
- 1-10 шт: 100 грн/шт
- 11-50 шт: 85 грн/шт
- 51-100 шт: 75 грн/шт
- 100+ шт: 70 грн/шт

Календар настільний:
- 1-10 шт: 60 грн/шт
- 11-50 шт: 50 грн/шт
- 51-100 шт: 45 грн/шт
- 100+ шт: 40 грн/шт
```

### Облік замовлень в друкарню
